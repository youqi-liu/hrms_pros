## 📌 项目选题说明

### 选题背景

随着企业规模的不断扩大，传统的人力资源管理方式已经无法满足现代企业的需求。本系统旨在通过信息化手段，提高人力资源管理的效率和准确性，为企业提供一站式的人力资源管理解决方案。

### 研究意义

**理论意义：**
- 验证前后端分离架构在企业级应用中的可行性
- 探索Spring Security在权限管理中的应用
- 研究Vue.js框架在复杂业务场景中的性能优化

**实践意义：**
- 提高企业人力资源管理效率，降低管理成本
- 实现数据的集中管理和共享，提升决策科学性
- 规范企业人事流程，提高员工满意度

### 项目创新点

1. **前后端分离架构**：采用Spring Boot + Vue.js，提高开发效率和系统可维护性
2. **细粒度权限控制**：基于Spring Security实现角色-权限-菜单的动态绑定
3. **数据可视化**：使用AntV G2Plot实现数据统计图表展示
4. **缓存优化**：使用Redis缓存热点数据，提升系统响应速度
5. **日志审计**：完整的操作日志记录，便于问题追溯

## 💡 技术实现要点

### 后端技术要点

#### 1. Spring Boot 自动装配
```java
// 启动类示例
@SpringBootApplication
public class LypApplication {
    public static void main(String[] args) {
        SpringApplication.run(LypApplication.class, args);
    }
}
```

**答辩说明：**
- `@SpringBootApplication` 是组合注解，包含了自动配置、组件扫描等功能
- Spring Boot 通过 SPI 机制实现自动装配

#### 2. Spring Security 安全框架
```java
// 配置类示例
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    // 配置HTTP安全策略
    // 配置认证管理器
    // 配置密码编码器
}
```

**答辩说明：**
- 使用过滤器链实现请求拦截
- BCrypt 算法进行密码加密
- JWT Token 实现无状态认证

#### 3. MyBatis Plus 持久层
```java
// Mapper 接口示例
public interface UserMapper extends BaseMapper<User> {
    // 继承BaseMapper，自动拥有CRUD方法
}
```

**答辩说明：**
- 使用代码生成器提高开发效率
- 条件构造器实现动态SQL
- 逻辑删除避免数据物理删除

#### 4. Redis 缓存
```java
// 缓存示例
@Cacheable(value = "user", key = "#id")
public User getUserById(Long id) {
    return userMapper.selectById(id);
}
```

**答辩说明：**
- 使用Redis缓存热点数据，减少数据库访问
- 设置合理的过期时间，避免缓存雪崩
- 缓存更新策略：先更新数据库，再删除缓存

### 前端技术要点

#### 1. Vue 组件化开发
```vue
<template>
  <div class="user-list">
    <!-- 组件模板 -->
  </div>
</template>

<script>
export default {
  name: 'UserList',
  data() {
    return {
      // 组件数据
    }
  },
  methods: {
    // 组件方法
  }
}
</script>
```

**答辩说明：**
- 组件化开发提高代码复用性
- 单文件组件(.vue)包含模板、逻辑、样式
- 父子组件通信使用 props 和 events

#### 2. Vuex 状态管理
```javascript
// store/index.js
export default new Vuex.Store({
  state: {
    user: null,
    token: ''
  },
  mutations: {
    SET_USER(state, user) {
      state.user = user
    }
  },
  actions: {
    login({ commit }, credentials) {
      // 登录逻辑
    }
  }
})
```

**答辩说明：**
- 集中式状态管理，便于数据共享
- mutations 同步更改状态
- actions 处理异步操作

#### 3. Vue Router 路由管理
```javascript
// router/index.js
const routes = [
  {
    path: '/user',
    component: Layout,
    children: [
      {
        path: 'list',
        component: UserList,
        meta: { requiresAuth: true }
      }
    ]
  }
]
```

**答辩说明：**
- 前端路由实现单页应用
- 路由守卫进行权限控制
- 动态路由加载提高安全性

#### 4. Axios 网络请求
```javascript
// api/index.js
export const getUserList = (params) => {
  return axios.get('/api/v1/users', { params })
}
```

**答辩说明：**
- 统一封装网络请求
- 请求拦截器添加Token
- 响应拦截器统一处理错误

## ❓ 常见答辩问题

### 基础问题

**Q1: 为什么选择Spring Boot框架？**
```
答：
1. 简化配置：通过自动装配减少XML配置
2. 内嵌服务器：无需单独部署Tomcat
3. 生产就绪：提供健康检查、监控等功能
4. 生态完善：与Spring全家桶无缝集成
5. 开发效率高：约定优于配置，快速启动项目
```

**Q2: 为什么采用前后端分离架构？**
```
答：
1. 职责分离：前端专注UI，后端专注业务逻辑
2. 并行开发：前后端可同时开发，提高效率
3. 易于扩展：可以为不同端提供统一的API
4. 性能优化：前端可单独部署到CDN
5. 技术选型灵活：前后端技术栈可独立演进
```

**Q3: 如何保证系统安全性？**
```
答：
1. 认证授权：Spring Security + JWT Token
2. 密码加密：BCrypt算法加密存储
3. SQL注入防护：MyBatis预编译
4. XSS防护：前端输入过滤
5. HTTPS传输：生产环境启用SSL
6. 接口鉴权：每个接口都需要Token验证
7. 日志审计：记录所有敏感操作
```

### 技术问题

**Q4: 如何解决高并发问题？**
```
答：
1. 缓存机制：使用Redis缓存热点数据
2. 数据库优化：添加索引，优化SQL
3. 连接池：使用Druid连接池管理数据库连接
4. 异步处理：耗时操作使用消息队列异步处理
5. 负载均衡：部署多个实例，使用Nginx负载均衡
```

**Q5: 数据库如何设计的？**
```
答：
1. 遵循三范式：减少数据冗余
2. 逻辑删除：使用deleted字段标记删除状态
3. 创建时间/更新时间：自动记录数据变更
4. 索引优化：为常用查询字段添加索引
5. 外键关联：使用外键保证数据完整性
```

**Q6: 如何实现权限控制？**
```
答：
1. RBAC模型：用户-角色-权限三层结构
2. 菜单权限：根据角色动态加载菜单
3. 按钮权限：使用自定义指令控制按钮显示
4. 数据权限：在SQL中添加数据权限过滤条件
5. 接口权限：通过注解标记接口所需权限
```
